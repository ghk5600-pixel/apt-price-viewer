# 관심단지 실거래 모니터

개인용 아파트 실거래 모니터링 웹앱입니다.

## 실행 방법

카카오맵과 장소검색을 사용하려면 `file://`가 아니라 `http://localhost:8080`으로 열어야 합니다.
`start-localhost-8080.cmd`를 실행한 뒤 아래 주소로 접속합니다.

```text
http://localhost:8080
```

## 현재 구현된 것

- 단지명/주소 검색
- 카카오 장소검색 기반 단지 후보 표시
- 관심단지 등록/해제 토글
- 실제 카카오맵 표시
- 카카오 좌표를 법정동코드로 변환
- 국토부 공동주택 단지 목록 API로 K-APT 단지명/코드 매칭
- 국토부 공동주택 기본 정보 API로 세대수, 동수, 준공년월, 주차대수 조회
- 관심단지 비교표
- 카카오 검색 단지의 최근 24개월 국토부 실거래 조회
- 데이터 범위 선택: 최근 6/12/18/24개월
- 59/74/84/101/114타입 면적 그룹핑
- 최근 실거래가, 3개월/6개월 중앙값, 평당가, 선택 기간 변화율
- 거래수 부족 여부 표시
- 국토부 실거래 API 인증키 설정/변경 메뉴
- 카카오맵 JavaScript 키 설정/변경 메뉴
- localhost 실행 시 실제 카카오맵 SDK 연결 시도
- 카카오 장소검색 결과를 관심단지로 등록
- Cloudflare Pages Functions용 `/api/...` 프록시 구조

## API 키 변경

로컬 실행에서는 상단 우측의 `API 설정` 버튼을 눌러 국토부 인증키와 카카오맵 JavaScript 키를
저장하거나 삭제할 수 있습니다. 키는 현재 브라우저의 `localStorage`에 저장됩니다.

Cloudflare 배포 환경에서는 국토부 인증키를 브라우저에 저장하지 않고,
Cloudflare Pages secret `MOLIT_SERVICE_KEY`에 저장합니다. 브라우저는 `/api/...`
경로만 호출하고, Pages Functions가 국토부 API를 대신 호출합니다.

인증키가 만료되어 새로 발급받은 경우:

```text
API 설정 클릭
→ 새 일반 인증키 입력
→ 저장
```

현재 연결 대상으로 잡아둔 API 엔드포인트는 아래와 같습니다.

```text
https://apis.data.go.kr/1613000/RTMSDataSvcAptTrade
https://apis.data.go.kr/1613000/AptListService3
https://apis.data.go.kr/1613000/AptBasisInfoServiceV4
```

카카오맵 JavaScript SDK는 아래 주소로 불러옵니다.

```text
https://dapi.kakao.com/v2/maps/sdk.js
```

카카오 개발자 콘솔에서 아래 항목이 필요합니다.

```text
플랫폼 키 > JavaScript 키 > JavaScript SDK 도메인: http://localhost:8080
카카오맵 > 사용 설정: ON
```

## 단지명 검색

`http://localhost:8080`으로 실행하고 카카오맵 SDK가 정상 로드되면 검색창에 입력한
단지명으로 카카오 장소검색 결과가 함께 표시됩니다.

```text
단지명 입력
→ 카카오 장소검색 후보 표시
→ 관심 버튼 클릭 또는 등록됨 버튼 재클릭
→ 관심단지 등록/해제
```

카카오 검색으로 등록한 단지는 좌표를 기준으로 법정동코드를 확인한 뒤,
국토부 공동주택 단지 목록 API에서 같은 법정동의 K-APT 단지명을 먼저 매칭합니다.
그 다음 공동주택 기본 정보 API에서 세대수, 동수, 준공년월, 주차대수를 조회하고,
최근 24개월의 국토부 아파트 매매 실거래 API를 조회합니다.
K-APT 단지명, 카카오 단지명, 지번이 매칭되면 거래수, 면적 그룹, 최근가, 중앙값, 평당가가 표시됩니다.
매칭되지 않는 세대수와 준공연도는 아직 `확인 필요`로 표시됩니다.

가운데 `관심단지 비교` 표의 `해제` 버튼으로도 관심단지를 삭제할 수 있습니다.

## 다음 단계 설계

국토부 실거래 API 서비스키는 브라우저에 직접 넣지 않는 것을 권장합니다.
현재는 Cloudflare Pages Functions용 API 프록시가 추가되어 있습니다.

```text
브라우저
→ /api/apt-list
→ /api/apt-basis
→ /api/building-ledger
→ /api/rtms
→ Cloudflare Pages Functions에서 국토부 API 호출
→ 화면에는 정리된 데이터만 반환
```

카카오맵 JavaScript 키는 프론트에서 사용해도 되지만, 카카오 개발자 콘솔에서
허용 도메인을 반드시 제한해야 합니다.

배포 순서는 `DEPLOY_CLOUDFLARE.md`를 참고합니다.

## 데이터 구조 메모

관심단지는 최소한 아래 정보를 저장하는 것을 추천합니다.

```text
id
단지명
주소
법정동코드
지번
위도
경도
세대수
준공연도
관심등록 여부
마지막 갱신일
```

실거래는 원본 전용면적을 보존하고, 비교용 면적 그룹을 별도로 붙입니다.

```text
전용면적 원본: 84.97㎡
면적 그룹: 84타입
평당가: 거래금액 / (전용면적㎡ / 3.3058)
```

이렇게 해야 단지 3~4곳을 비교할 때 비슷한 면적끼리만 비교할 수 있습니다.
